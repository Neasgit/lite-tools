The Six Skills

One job each, no overlap (mostly)

LITE runs on six purpose-built agents. Each one handles a specific stage of the case lifecycle. The principle: one job per skill. If a skill needs to do two things, it's two skills.

Stage 1 · Discovery
lite-research
Investigate a new case from scratch. Searches news archives, court records, obituaries, and footage sources. Writes a structured Research Brief into the case folder.
Triggers: "research case X", "start research", "fill the brief"
Stage 2 · Verification
lite-factcheck
Verify the Research Brief before scripting. Cross-references every claim, flags contradictions, double-sources identity claims. Writes a Fact-Check Report with verdict.
Triggers: "fact-check", "verify", "is this confirmed", "sources"
Stage 3 · Script
lite-scriptwriter
Turn verified research into a full shooting script. Cold open, parts, outro, production notes. Uses the EWU documentary voice. Flags caption moments and footage gaps.
Triggers: "write the script", "draft narration", "script this up"
Stage 4 & 6 · Gates
lite-qa
Pre-flight production readiness check. Runs twice per case: pre-script-lock and pre-publish. Walks 12 editorial gates, grades each, names blockers.
Triggers: "QA case X", "ready to lock", "ready to ship", "pre-flight"
Stage 5 & 8 · Production
lite-edit
Premiere Pro, CapCut, and OpusClip expert. Answers any question about editing, captions, exports, motion graphics, B-roll, Shorts. Step-by-step for a beginner.
Triggers: "Premiere", "CapCut", "OpusClip", "timeline", "captions", "export"
Ongoing · Maintenance
lite-vault-mod
Audits the Obsidian vault for stale notes, format drift, duplicates, and orphaned files. Read-only. Produces a dated audit report; waits for approval before any changes.
Triggers: "audit the vault", "is anything stale", "tidy up"

The Full Case Lifecycle

Discovery to publish

Every case flows through the same eight stages. The agent column shows which skill drives that stage. Manual work (no skill) is greyed.

01
Discovery & Research lite-research
Pick a case from the pipeline. Invoke lite-research with case number and name. The agent searches news, court records, obituaries, footage sources.
→ Research Brief.md
02
Verification lite-factcheck
Run on the Brief. Independently re-verifies claims, flags contradictions, double-sources identity claims. Verdict: CLEAR / MINOR / HOLD. Don't script until at least MINOR.
→ Fact-Check Report.md
03
Script lite-scriptwriter
Reads Brief + Fact-Check + Footage Log. Writes a full shooting script with cold open, parts, outro, and production notes. Flags caption moments, footage gaps, and verify-before-lock items.
→ Script.md
04
Pre-Lock QA lite-qa (pre-lock)
Before recording voiceover. Walks 12 editorial gates: editorial standards, legal/ethical, sourcing, attribution, guilt framing, distress justification. Catches issues while the script is still editable.
→ QA Report — pre-lock.md
05
Production (Edit) lite-edit Premiere · CapCut
Record VO (Audacity → Adobe Podcast → WAV). Work the Five Version Pass in Premiere: V1 Assembly → V2 VO Sync → V3 Graphics → V4 Colour/Audio → V5 Captions (via CapCut roundtrip for the EWU yellow-accent look).
→ Edit master in Premiere
06
Pre-Publish QA lite-qa (pre-publish)
Final check before upload. Same 12 gates as pre-lock but with later-stage items now in scope: thumbnail, description, pinned comment, source attributions. Catches anything that slipped through production.
→ QA Report — pre-publish.md
07
Publish manual
Upload to YouTube. Paste description from Sources.md. Post the pinned VO-disclosure comment. Move the case folder from 01 - Cases/Active/ to 01 - Cases/Complete/.
→ Live on channel
08
Shorts (optional) lite-edit (OpusClip)
Feed the final master into OpusClip. AI generates 5-15 vertical clips. Review each one before publishing (don't trust AI selections blindly). Upload to YouTube Shorts, TikTok, Reels.
→ Short-form distribution
∞
Ongoing maintenance lite-vault-mod
Every few weeks (or after a significant case reframe), run a full vault audit. Catches stale notes, format drift, duplicate content, orphaned files. Approve fixes before they're applied.
→ Vault Audit — YYYY-MM-DD.md

04

When NOT To Use Each Skill

Wrong tool, wrong stage

Each skill has a clear "out of scope" boundary. Calling the wrong one wastes a session.

lite-research
  • Don't re-run if a recent brief already exists. Use lite-factcheck or lite-vault-mod.
  • Don't run on skipped or parked cases. Check Skip List first.
lite-factcheck
  • Don't run before research has produced a brief. Nothing to check.
  • Doesn't audit production readiness — that's lite-qa.
lite-scriptwriter
  • Don't run if Fact-Check Report says HOLD. Resolve blockers first.
  • Don't run without a Research Brief. Needs the factual foundation.
lite-qa
  • Don't run before the script is drafted. Most gates will be DEFERRED.
  • Doesn't fact-check claims. Assumes facts are already verified.
  • Doesn't edit files — read-only.
lite-edit
  • Not for research, scripting, or fact-checking.
  • Don't ask it for editorial decisions — it's a software helper, not an editor.
lite-vault-mod
  • Doesn't write changes unilaterally. Always produces an audit first.
  • Don't run on the same day twice. Audit reports clutter quickly.
05

How Skills Overlap

Keep neighbours distinct

A few skills have neighbouring jobs. These rules keep them clean.

lite-factcheck vs lite-qa. Factcheck is "are the facts right?" QA is "is the case ready?" Factcheck audits research. QA audits production readiness. Both are needed; neither replaces the other.
lite-vault-mod vs lite-qa. Vault-mod looks across the whole vault for stale notes and format drift. QA goes deep on one case's production readiness. Vault-mod periodically; QA at case milestones.
lite-research vs lite-factcheck. Research goes wide (find everything). Factcheck goes deep (verify what's been found). Always wide first, deep second.
lite-scriptwriter vs lite-edit. Scriptwriter writes the words. lite-edit helps execute them in the editor. Scripts have [CAP:] flags that lite-edit knows how to action in Premiere or CapCut.
