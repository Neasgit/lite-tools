Redaction / Reveal.

The visual language for cuts, omissions, and reveal moments. Six devices, all sharing one rule: charcoal hides, amber surfaces. Click any device to see it animate.

Version 1.0 Devices 6 Brand spec ·04 of Brand Reference
01 · Why this exists

The name Left in the Edit is about what gets cut, omitted, or hidden.

The brand needs a visual device that says the same thing: something was removed, something is being recovered, something is being revealed. That's what this system is for. Six devices. All work together. Amber is reserved for the reveal point · never the redaction itself.

02 · The six devices

Each shows a different kind of "left in the edit". Click any stage to see the reveal.

The structural workhorse. Use whenever the case turns on a contradiction.

06Page Break / Void

03 · Usage Rules

Devices share rules. Break these and the system stops reading as a system.

? Constants

  • Charcoal (#060606) is the redaction colour. It hides.
  • Amber (#D4A800) is the reveal colour. It surfaces.
  • JetBrains Mono is the type register for every device.
  • Reveals are paired with the sub-bass sting (Brand Reference ·04).
  • Each device gets its own beat · never two at once.

? Cadence

  • Amber Reveal Line · once per video, maximum.
  • Redaction Bar · used as often as the case warrants.
  • Cut Marker · paired with the cut log in the description.
  • Before / After · for the case-turning contradiction.
  • Document Highlight · only on a line that actually turns the case.
  • Page Break / Void · for genuine omissions, not absence-as-effect.
